## What to build

### 1. Launch Step (end of /create wizard)
Complete the final wizard step that connects creation to the marketplace:
- Mint price input (XCH amount)
- Launch date/time picker (or "launch immediately" toggle)
- Allowlist textarea (one xch1 address per line, optional)
- Reveal type toggle: Instant (buyer sees their NFT immediately) or Blind (all NFTs show placeholder until creator reveals)
- "Publish Collection" button → creates marketplace listing in Supabase, sets project status to "live" or "scheduled"
- After publish: redirect to /marketplace/:id/manage

### 2. Marketplace Browse Page (/marketplace)
- Grid of published collections (status = live or scheduled)
- Each card shows: collection image (first generated NFT or placeholder), name, mint price, X/Y minted, launch countdown if scheduled
- Filter bar: All / Live Now / Upcoming / Sold Out
- Search by collection name
- Clicking a card goes to /marketplace/:id

### 3. Collection Mint Page (/marketplace/:id)
- Full-width hero (first NFT image or video)
- Collection name, description, creator address
- Supply progress: "247 / 1000 minted" with progress bar
- Mint price prominently displayed in XCH + USD equivalent
- Countdown timer if not yet live (flips to Mint button at launch time)
- Allowlist badge if wallet address is on allowlist
- MINT button → triggers payment flow:
  - Generate unique XCH payment address for this order
  - Show address + QR code + amount
  - "Waiting for payment..." with live status
  - On payment detected → "Minting your NFT..." spinner
  - On mint confirmed → show the NFT (or placeholder if blind)
- Gallery section: grid of already-minted NFTs from this collection
- Stats: floor price, total volume, holders count

### 4. Minting Engine (backend)
- POST /api/marketplace/:id/orders — create order, generate payment address
- GET /api/marketplace/:id/orders/:orderId — order status
- Payment watcher: polls Chia node every 10s for incoming XCH to payment addresses
- On payment confirmed: queue BullMQ mint job
- Mint job: 
  - Select next unminted token from generated_tokens table (random or sequential based on creator setting)
  - Build Chia NFT1 spend bundle using wallet daemon nft_mint_nft
  - Submit to node
  - Poll for confirmation
  - Update order status to confirmed
  - Update generated_tokens row with buyer address
- Idempotency: order has unique payment_address, mint job checks if already minted before proceeding
- Duplicate prevention: lock token row during minting (Postgres SELECT FOR UPDATE)

### 5. Creator Management Page (/marketplace/:id/manage)
- Only accessible if wallet address = project creator_address
- Live stats: mints today, total revenue in XCH, remaining supply
- Orders table: recent orders with status (pending payment / minting / confirmed / failed)
- Pause/Resume toggle for minting
- Manual mint form: enter a recipient address to gift an NFT
- Reveal button (if blind mint): reveals all NFTs simultaneously, updates metadata URIs on-chain
- Export: download CSV of all orders and recipient addresses
